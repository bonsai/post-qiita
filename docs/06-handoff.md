## 開発体験

### よかったこと

- **Qiita API v2 はシンプル**: エンドポイントが少なく、`POST /items` だけで完結する。認証も Bearer token のみ。
- **親子対話形式のハマり感**: 技術解説の敷居が下がる。会話の流れで自然に説明できる。
- **CLI ファーストの開発サイクル**: ブラウザを開かずに完結するので、エディタから離れずに済む。

### ハマったところ

- **Qiita API v2 は series 作成未サポート**: ブラウザで手動作成 → API で紐付けの二段階が必要。
- **403 Forbidden と scope**: `write_qiita` がないと 403。エラーメッセージが曖昧なので注意。
- **重複投稿の扱い**: 同じ内容を POST すると 403 になるケースがある。既存記事は PATCH で更新する設計が安全。
- **親子対話の最後のセリフ**: 過去記事で「そういう時代だね😎」を連発しすぎた。以後は内容に合わせた独自の締めにしている。

### 運用のコツ

- 新規記事は必ず `--draft` で下書き投稿してから Qiita 上で最終確認
- シリーズは全話書き終えてから一気に投稿すると構成の一貫性が保ちやすい
- `sync-status.mjs` を定期的に実行してローカルと Qiita のズレを検知
