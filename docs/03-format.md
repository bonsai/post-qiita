# Markdown フォーマット

## 基本構造

```markdown
---
tags: [タグ1, タグ2, タグ3]
---

# 記事のタイトル

本文...
```

## YAML Front Matter

記事先頭を `---` で囲む。

| フィールド | 必須 | 説明 |
|-----------|------|------|
| `tags` | ✅ | 配列形式 `[タグ1, タグ2]`。空の場合は `tags: []` |
| `id` | — | 記事の独自ID（例: `godot-pwa-03`）。sync-status で状態管理に使用 |

## タイトル抽出ルール

1. 本文の最初の `# `（H1）がタイトル
2. H1 がない場合 → ファイル名（拡張子除く）がタイトル（非推奨）

```markdown
---
tags: [Qiita, CLI]
---

# ここがタイトルになる    ← 最初の H1

このテキストが本文
```

## タグ

```yaml
tags: [Qiita, CLI, Node.js, 自動化]

# スペースを含む場合
tags: ["Web開発", "CI/CD"]
```

## 親子対話スタイル

```
---
tags: [Qiita, CLI, 親子対話]
---

# 親子対話：タイトル

父「最初のセリフ」

娘「応答」
```

- 父・娘の会話形式で技術解説
- **各記事の最後のセリフは内容に合わせた独自のものにする**

## 独自 ID による状態管理

Front matter に `id` フィールドを追加することで、sync-status がローカルと Qiita の対応を追跡できる。

```yaml
---
tags: [ゲーム開発, Godot, PWA]
id: godot-pwa-03
---
```

ID ありの記事は Qiita の記事 ID と紐付けて管理。ID なしの記事はファイル名ベースで表示される。

## ファイル命名

`wiki/blog/tech/godot-pwa/` 以下の連載は `NN-article-name.md` 形式。

## よくあるエラー

| 症状 | 原因 |
|------|------|
| 422 | `tags` 未指定・空配列・タイトルなし・本文空 |
| タイトル未認識 | H1 の前に空行がある |
